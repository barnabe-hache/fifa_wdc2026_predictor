# 🏆 FIFA World Cup 2026 Score Predictor

A machine learning system that predicts match scores for the FIFA World Cup 2026, combining historical match data with FIFA player ratings to generate calibrated probabilistic forecasts — exact scores, win/draw/loss probabilities, expected goals, and more — through an interactive web dashboard.

## Overview

This project frames football score prediction as a **dual count-regression problem**: rather than predicting a single "winner," it models the number of goals each team is expected to score as two independent Poisson-distributed variables. This statistical framing — standard in football analytics — makes it possible to derive a full probability distribution over every possible scoreline, not just a point estimate.

The pipeline trains an ensemble of two complementary models on over a century of international match results, enriches it with FIFA player-rating data and a custom-built dynamic ELO system, and serves predictions through a Streamlit dashboard designed for non-technical end users.

---

## Key features

- **Dual-model ensemble** — a Poisson GLM and a LightGBM regressor (Poisson objective) trained independently per team side, blended and calibrated for accuracy.
- **Dynamic ELO ratings** — recomputed match-by-match across the full historical dataset, with an adaptive K-factor and goal-difference multiplier.
- **Neutral-venue symmetry** — a custom data-augmentation and double-inference scheme guarantees that `predict(A, B)` and `predict(B, A)` are mathematically identical on neutral ground, eliminating a common bias in home/away-structured models.
- **Rich feature set** — rolling form (5/10/20-match windows), exponentially-weighted recent form, head-to-head history, home-advantage factors, and FIFA squad-quality metrics (overall rating, market value, per-position attributes).
- **Calibrated outputs** — Isotonic Regression corrects systematic bias before the Poisson distribution is applied, improving the reliability of downstream probabilities.
- **20+ derived metrics per prediction** — exact score, outcome probabilities, full score matrix, Over/Under and BTTS markets, clean-sheet probabilities, dominance index, upset probability, and more — exposed through a single function call.
- **Interactive dashboard** — country search with flags, neutral/home-venue toggle, tabbed metric breakdown, hover tooltips explaining every statistic, and a dedicated methodology page.

## How it works

### 1. Feature engineering

Each historical match is enriched with:

- **Dynamic ELO** — ratings are recomputed sequentially across the entire match history (from the 19th century onward), with a K-factor that scales with competition importance (World Cup matches carry roughly twice the weight of friendlies) and a goal-difference multiplier inspired by the Dixon-Coles approach.
- **Rolling form** — goals scored/conceded, win rate, and clean-sheet rate over trailing 5/10/20-match windows, plus an exponentially-weighted moving average (span = 10) that gives more weight to the most recent matches without a hard cutoff.
- **Head-to-head** — outcome and goal statistics from the last 10 direct meetings between the two sides.
- **Home-advantage factor** — the per-team delta in scoring and win rate between home matches and neutral-venue matches, capturing the fact that some nations benefit far more from home support than others.
- **FIFA strength** — squad-quality aggregates (top-11/23/5 overall ratings, market value, and per-attribute means for shooting, passing, defending, pace, physicality, and dribbling).

### 2. Model training

Two model families are trained independently for home goals and away goals:

| Model | Role |
|---|---|
| Poisson GLM (`sklearn.PoissonRegressor`) | Linear, highly interpretable, robust on smaller samples |
| LightGBM (`objective=poisson`) | Captures non-linear interactions (e.g. a strong attack against a weak defence is not simply additive) |

The two predictions are combined into a weighted ensemble (60% LightGBM, 40% GLM), then passed through an `IsotonicRegression` calibrator fitted on the training set to correct the systematic over-prediction of home goals observed in the raw ensemble.

A strict **temporal train/test split** is used for evaluation (the most recent 15% of matches are held out) to avoid information leakage from the future.

### 3. Neutral-venue symmetry

Football models trained on home/away-labeled data tend to learn a structural "home" bias even when matches are actually played on neutral ground — a known pitfall for World Cup-style tournaments. Two complementary mechanisms eliminate this:

- **Training-time augmentation**: every neutral-venue match in the training set is duplicated with home/away teams and features swapped, teaching the model that side assignment is arbitrary on neutral ground.
- **Inference-time double pass**: at prediction time, both `(A, B)` and `(B, A)` are evaluated, and the resulting lambdas are averaged — `λ_A = mean(λ_home(A,B), λ_away(B,A))`. This guarantees `predict_score(A, B) == predict_score(B, A)` by construction whenever the venue is neutral.

### 4. Probabilistic inference

The calibrated expected-goal values (λ_home, λ_away) parameterize two independent Poisson distributions, from which a full **9×9 score probability matrix** is built: `P(home = i, away = j)` for every combination of `i, j ∈ [0, 8]`. Every output metric — win/draw/loss probabilities, the most likely exact score, Over/Under markets, Both-Teams-To-Score probability, clean-sheet probability — is a direct mathematical derivation from this matrix, ensuring all reported figures are internally consistent.

### 5. Inference API

The entire system is exposed through a single function:

```python
from predict import predict_score

result = predict_score("France", "Brazil", neutral=True, tournament="FIFA World Cup")

result["prediction"]          # {'home': 2, 'away': 1}
result["result_probs"]        # {'home_win': 0.41, 'draw': 0.26, 'away_win': 0.33}
result["xg_analysis"]         # {'xg_home': 1.74, 'xg_away': 1.21, 'btts_prob': 0.64, ...}
result["elo_home"]            # 1923.4
result["dominance_index"]     # 0.187
result["h2h"]                 # {'h2h_n': 12, 'h2h_home_winrate': 0.42, ...}
```

`predict_score()` lazily loads and caches all model artifacts on first call, making repeated predictions (as used in the dashboard) essentially free after warm-up.

## Model performance

Evaluated on a strict temporal hold-out (the most recent 15% of matches, never seen during training):

| Metric | Result | Naïve baseline |
|---|---|---|
| Home goals MAE | ≈ 1.05–1.10 | — |
| Correct outcome (W/D/L) | ≈ 64% | ~50% |
| Correct exact score | ≈ 12% | ~7% (random) |

Football is inherently high-variance — exact-score prediction is a genuinely hard problem even for professional analysts — but the model delivers a consistent ~14-point lift over a naïve baseline on outcome prediction, and produces well-calibrated probability estimates suitable for downstream analysis.

## Running locally

```bash
git clone <repo-url>
cd prono_cdm_2026
pip install -r requirements.txt

# Train the models (only needed once, or to refresh on new data)
python train.py

# Run the dashboard
streamlit run streamlit_app.py
```

The app expects pre-trained artifacts in `models/`. If you only want to explore the dashboard, the repository ships with the trained models included — no training step required.

```bash
python predict.py "France" "Brazil"
python predict.py "Argentina" "Germany" --no-neutral
```

## Design notes

- **No hardcoded paths** — all file locations are resolved relative to the script's own location (`Path(__file__).resolve().parent`), making the project portable across local machines, CI, and Streamlit Cloud without configuration.
- **Lightweight by design** — both models were deliberately chosen for low training and inference cost (a few seconds to retrain on the full dataset, sub-100ms inference), making the project easy to redeploy or extend without specialized infrastructure.
- **Single source of truth for metrics** — every probability displayed in the dashboard is derived from the same score matrix returned by `predict_score()`, so the UI can never display inconsistent figures.
